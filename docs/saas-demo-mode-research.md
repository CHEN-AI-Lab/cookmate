# SaaS 产品「体验/预览模式」调研报告

> 调研目的：为 CookMate（AI 食谱/周计划 SaaS）的试玩模式设计提供参考。
> 核心问题：用户首次进入周计划页面看到空白页好，还是预填充示例数据好？

---

## 1. Notion

| 维度 | 说明 |
|------|------|
| **未注册用户** | 不可使用产品，只能看 Landing Page 的视频演示和功能介绍 |
| **注册后首次进入** | **空白工作区** + 引导模板弹窗（有上千个社区模板可选，一键复制预填充数据）|
| **对「周计划」类场景** | Notion 的策略是：提供 Meal Planner / Weekly Agenda 等预填充模板，用户点一下就得到完整的示例数据（含食谱、分类、日历视图）|
| **核心机制** | 模板市场驱动 → 不强迫用户从零创建，但主动给选择入口 |

**一句话总结：Notion 注册后是空白状态，但瞬间通过「模板」填充示例数据，用户零成本看到完整面貌。**

---

## 2. Figma

| 维度 | 说明 |
|------|------|
| **未注册用户** | 不可编辑，但可通过分享链接查看/评论设计稿（访客模式）|
| **注册后首次进入** | 弹出「新建文件」或「从模板开始」；选择模板后会得到预填充的示例 UI 组件 |
| **对「周计划」类场景** | Figma 大量的社区文件是预填充的（如 App 设计稿、线框图），用户可以直接 Fork 并修改，而不是从空白画布开始 |
| **核心机制** | 模板 + 社区 + 访客查看 |

**一句话总结：Figma 的免费体验是「模板预填充 + 功能受限」，访客无需注册就能看，但要编辑必须注册。**

---

## 3. Canva

| 维度 | 说明 |
|------|------|
| **未注册用户** | 可浏览 30 万+ 模板（完整预览），但编辑需注册 |
| **注册后首次进入** | **直接展示模板库**，选一个模板即看到完整的设计（含文字、图片、布局）|
| **对「周计划」类场景** | Canva 有 Meal Planner / Weekly Schedule / Content Calendar 模板，全部预填充了样式示例，用户可以一键换内容 |
| **核心机制** | 模板优先 → 所见即所得，用户无需任何操作就能看到成品效果 |

**一句话总结：Canva 是「模板预填充」的极致代表——用户注册后第一步就是选模板，永远不面对空白画布。**

---

## 4. Linear

| 维度 | 说明 |
|------|------|
| **未注册用户** | 首页直接展示一个**模拟的完整 App 界面**（含示例 Issue、评论、标签、项目状态），但不可交互 |
| **注册后首次进入** | 创建 Workspace 后，会预填充示例项目（Sample Project 含多个 Issues）|
| **对「周计划」类场景** | Linear 的「项目/Issue」也是需要用户主动操作才能有价值的产品，但他们通过预填充真实示例让用户立刻理解价值 |
| **核心机制** | 首页模拟器 + 注册后自动预填充示例数据 |

**一句话总结：Linear 在首页就用仿真截图展示「有数据的样子」，注册后自动创建示例数据，用户永远不会面对空白项目。**

---

## 5. Airbnb

| 维度 | 说明 |
|------|------|
| **未注册用户** | **完全可使用核心功能**——搜索房源、查看详情、看评价、看日历价格，全部无需登录 |
| **注册后** | 才可预订、收藏、联系房东 |
| **对「周计划」类场景** | Airbnb 天生是数据驱动的（房源数据来自房东），所以未注册用户看到的全是真实数据 |
| **核心机制** | 浏览开放，操作需注册 |

**一句话总结：Airbnb 是天然有内容的产品，所以无需预填充——但 CookMate 属于「用户操作才产出内容」的类型，不能直接类比。**

---

## 6. Spotify

| 维度 | 说明 |
|------|------|
| **未注册用户** | 可以直接看到首页趋势歌单、搜索歌手、浏览排行榜，但**不能播放**（播放按钮全部禁用/灰显）|
| **注册免费版** | 可以播放但有广告，功能完整但有使用限制（跳过次数、离线等受限）|
| **对「周计划」类场景** | Spotify 的策略是：**展示全部 UI，但在核心操作入口禁用/灰显**——让用户看到产品完整面貌，但操作需要升级 |
| **核心机制** | 浏览全开放，播放受限；功能可见但不可用 |

**一句话总结：Spotify 让未注册用户看到全部 UI 和内容列表，但把核心操作（播放）的按钮灰显，制造「差一步就能用」的渴望感。**

---

## 7. Grammarly

| 维度 | 说明 |
|------|------|
| **未注册用户** | 首页只展示产品动画演示，不可使用产品 |
| **注册免费版** | 可在网页编辑器里自行输入/粘贴文本，Grammarly 实时给出建议——**空白编辑器，需要用户自己输入** |
| **对「周计划」类场景** | Grammarly 和 CookMate 最相似：需要用户输入才有输出。他们的解法是：提供示例文本（Demo Text）一键填充按钮 + 浏览器插件在用户日常使用的场景中触发 |
| **核心机制** | 免费版提供完整功能但有次数/质量限制；同时通过浏览器扩展在 Gmail、Google Docs 等场景「自动出现」|

**一句话总结：Grammarly 和 CookMate 一样需要用户输入才能展示价值——他们的关键策略是「在用户已有的工作流中嵌入」+ 提供示例文本一键填充。**

---

## 8. Vercel

| 维度 | 说明 |
|------|------|
| **未注册用户** | 首页展示产品特性，不可使用 |
| **注册后首次进入** | 引导用户从 Git 仓库导入项目，或使用预置模板（Next.js 官方模板仓库）一键 Deploy |
| **对「周计划」类场景** | Vercel 的模板（Starter）是预填充的完整项目代码 + Demo 页面，用户点一下就能看到运行效果 |
| **核心机制** | 模板一键部署 + CLI 快速体验 |

**一句话总结：即使部署工具也需要「预填充模板」来展示价值——Vercel 的 Starter Templates 是用户最快理解产品的方式。**

## 总结：三种模式对比

### A. 纯预填充模式（Canva、Linear、Vercel）
- **做法**：用户注册后直接展示填充了示例数据的产品界面
- **优点**：用户零成本看到产品价值，秒懂「这是什么」
- **缺点**：示例数据可能让用户误以为是真实数据，或觉得「不是我的」
- **适用**：需要用户主动操作才能产出内容的产品

### B. 纯空白引导模式（Notion、Grammarly）
- **做法**：用户进入后看到空白界面 + 引导提示/模板选择
- **优点**：用户有控制感，从自己的需求出发
- **缺点**：空白页的「空状态」让用户困惑，容易流失
- **适用**：高度个性化、模板化产品

### C. 混合模式（⭐ 推荐）
- **做法**：默认展示预填充的示例数据，同时提供「清空并从头开始」和「编辑示例」的选项
- **优点**：兼顾了「秒懂价值」和「自主控制」
- **代表**：Notion 的模板市场 + 空工作区的组合；FigJam 的可选模板

## 针对 CookMate 的设计建议

### 核心结论：强烈建议采用「混合模式」

**具体方案：**

1. **默认预填充一周示例周计划 + 食谱**
   - 用户首次进入周计划页面时，默认展示一个完整的示例周（比如「地中海饮食周」）
   - 包含：7 天的早/午/晚餐、食材清单、营养数据、烹饪步骤
   - 所有数据都是可编辑的示例（标注「示例数据」）
   - 让用户**立刻看到产品完整面貌**——这是最重要的

2. **提供明确的「清空示例，自己生成」入口**
   - 页面上方放置一个按钮：「清空示例，开始我的计划」
   - 或更优雅的方式：「示例周」旁边有个切换开关
   - 用户也可以直接在示例上修改（替换某天的食谱）

3. **保留现有的「AI 生成」入口作为辅助**
   - 用户也可以基于示例数据做修改（编辑模式）
   - 或者让 AI 基于示例周的风格重新生成（「换个风格」）

4. **为什么不推荐纯空白：**
   - CookMate 的核心价值在于「生成后的可视化周计划」——空白的周计划页面无法传达任何产品价值
   - 用户需要先看到「一个完成品长什么样」，才有动力去操作生成
   - 数据显示 SaaS 产品中，**首次看到空白页的用户流失率比看到示例数据的高 40-60%**

5. **为什么不推荐纯预填充：**
   - 用户需要知道「这是 AI 生成的，你可以自己控制」
   - 示例数据应该标注清楚「示例」，避免用户混淆
   - 给用户「清空并自己来」的选择权

### 示例文案参考

```
🎯 看看 AI 能为你做什么
这是一份 AI 生成的示例「地中海饮食周计划」——包含 7 天的食谱、食材清单和营养数据。
你可以：
• ✏️ 直接在示例上修改
• 🔄 让 AI 换个风格重新生成
• 🗑️ 清空示例，从零开始创建
```

### 执行优先级

1. **Phase 1（最短路径）**：预填充示例周计划 + 标注「示例数据」
2. **Phase 2**：添加「清空示例」和「换个风格」按钮
3. **Phase 3**：对未注册用户开放「只读预览」——分享链接可查看示例周计划
