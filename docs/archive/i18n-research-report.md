## 调研结论

### 一、目前已有的基础

项目已使用 **next-intl**，这是 Next.js 生态最主流、最成熟的国际化方案（GitHub 4.3k stars）。它本身支持**回退机制**——某个 key 在目标语言没有翻译时，自动显示英文，不报错、不崩溃。

### 二、四种方案对比

#### 方案 A：AI 自写翻译脚本（推荐）

**做法：** 一个 `scripts/translate.mjs`，读 `en.json`，调用 AI 翻译缺失的 key，写回目标语言文件。

**优点：**
- 免费——只花 AI API 费用（gpt-4o-mini 翻译 600 key 约 ¥0.10）
- 完全可控——翻译质量、术语、格式自己定
- 无第三方依赖——不需要注册任何 SaaS 平台
- 增量翻译——新增 key 只翻译新增的，已有的不动
- 适合现有架构——next-intl + JSON 翻译文件直接适配

**缺点：**
- 需要自己维护脚本
- 没有可视化界面
- 没有翻译记忆库

#### 方案 B：Tolgee（开源，可自托管）

**官网：** tolgee.io
**定价：** 免费版 500 key / 3 席位；付费 €49-€499/月
**开源：** 可自托管（免费，无限 key）
**支持 next-intl：** ✅ 有 MCP 服务器，支持 JSON 格式

**优点：** 可视化界面、翻译记忆库、AI 自动翻译
**缺点：** 自托管需额外维护，免费 SaaS 版 500 key 不够用

#### 方案 C：Crowdin / Lokalise / POEditor（SaaS 平台）

| 平台 | 免费版 | 付费版 |
|------|--------|--------|
| Crowdin | 1 项目 / 不限 key | $45+/月 |
| Lokalise | 14 天试用 | $144-$999+/月 |
| POEditor | 14 天试用 | €15-€49/月 |

**优点：** 专业级翻译管理、翻译记忆、术语库
**缺点：** 贵、学习成本、绑定第三方

#### 方案 D：纯人工翻译

**优点：** 最精准
**缺点：** 极其耗时、容易遗漏、不可扩展

### 三、详细对比表

| 对比项 | AI 脚本 | Tolgee 自托管 | SaaS 平台 | 人工翻译 |
|--------|:-------:|:-----------:|:---------:|:-------:|
| 费用 | ¥0.10/次 | 服务器费 | $15-999/月 | 时间成本 |
| 前期配置 | 1 小时 | 1-2 天 | 1-2 天 | 0 |
| 新增语言 | 1 分钟 | 几分钟 | 几分钟 | 几小时 |
| 增量翻译 | ✅ 自动 | ✅ 自动 | ✅ 自动 | ❌ 手动 |
| 可视化界面 | ❌ | ✅ | ✅ | ❌ |
| 翻译记忆 | ❌（可加） | ✅ | ✅ | ❌ |
| 第三方依赖 | ❌ | 自己的服务器 | 外部平台 | ❌ |
| 学习成本 | 低 | 中 | 中 | 低 |
| 可扩展性 | 无限 | 无限 | 按套餐 | 有限 |

### 四、推荐方案

**推荐：方案 A（AI 脚本）+ 增量改进**

理由：
1. 项目已用 next-intl + JSON 文件，脚本直接适配现有架构
2. 零持续成本，每次翻译几分钱
3. 未来所有项目都能用，同一个脚本复制到新项目就能跑
4. 风险最低，不依赖任何第三方

**以后如需升级：** 可无痛切换到方案 B（Tolgee 自托管），翻译文件可直接导入，无需改代码。

### 五、脚本不足与改进

| 不足 | 改进方案 |
|------|---------|
| 没有翻译记忆 | 加缓存文件记录已翻译原文 |
| 没有可视化界面 | 加简单 Web 页面列出待翻译 key |
| 一次翻译一个语言 | 加批量翻译多种语言 |

**当前规模（2 种语言、600 key）下脚本已够用，不需要立即改进。**

### 六、结论

**不需要推翻重来。** 现有 `next-intl` + JSON 翻译文件架构完全正确，是行业标准做法。`scripts/translate.mjs` 脚本是锦上添花，加新语言时省时间用的。

就算以后不用这个脚本，现有翻译文件（`en.json` / `zh-CN.json`）在任何平台（Tolgee / Crowdin / Lokalise）都能直接导入，不存在推翻重来的问题。
