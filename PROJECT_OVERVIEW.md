# CookMate 项目概览

## 核心理念

> 像有个人帮你管冰箱、想菜谱、做周计划 —— 原来每天花在"吃什么"上的 20 分钟，现在 2 秒钟解决。

## 完整数据流

```
用户买回来的食材
        │
        ▼
    ┌─────────────┐
    │  食材库      │  ← 用户手动录入或扫描
    │  (Pantry)    │
    └──────┬──────┘
           │
           ├──→ AI 菜谱页：食材库点"做菜"自动带入选中的食材
           │
           ├──→ 周计划生成：AI 优先使用食材库中已有的食材
           │
           └──→ 购物清单：已拥有的食材标"已有"，不重复加入购物清单
           │
           ▼
    ┌─────────────┐
    │  AI 菜谱     │  ← 输入食材 → AI 生成菜谱
    │  (Recipes)   │     （有 Key=正式AI，无Key=演示数据）
    └──────┬──────┘
           │
           └──→ 周计划：每个菜谱可点击"加入周计划"排入某天某餐
           │
           ▼
    ┌─────────────┐
    │  周计划      │  ← 一键 AI 生成整周 21 餐
    │  (Meal Plan) │     每个格子可手动编辑(改菜名)
    └──────┬──────┘
           │
           └──→ 自动计算所需食材
           │
           ▼
    ┌─────────────┐
    │  购物清单    │  ← (周计划需要的食材) - (食材库已有的) = 该买的
    │  (Grocery)   │     按类别分组（蔬菜区、肉类区…）
    └─────────────┘
```

## 模块关联矩阵

| 操作 | 影响 |
|------|------|
| 食材库添加食材 | ✅ AI菜谱可用、周计划优先使用、购物清单标记已有 |
| 食材库删除食材 | ✅ 同上反向 |
| 食材库点"做菜" | ✅ 食材自动带到 AI 菜谱页 |
| AI 菜谱生成 | 无（菜谱页独立，生成的菜不自动进周计划） |
| AI 菜谱点"加入周计划" | ✅ 菜谱排入指定天的指定餐（早/午/晚） |
| 周计划生成 | ✅ 按周存入数据库，编辑改菜名也保存 |
| 周计划 → 购物清单 | ✅ 自动提取所有食材，按类别分组 |
| 购物清单勾选完成 | ✅ 标记完成状态 |

## 关键设计决策

- 所有按钮始终显示（不隐藏），未配置功能点击提示"上线后可用"
- 无 AI Key 时自动回退演示数据，所有功能可零配置本地测试
- 本地开发模式不限 AI 调用次数（生产环境免费用户每天 1 次）
- 食材风控：7 大类 80+ 词黑名单 + AI prompt 双保险，拒绝非食用/保护/虚构食材

## 技术栈

- Next.js 14 (App Router) + TypeScript + Tailwind CSS
- SQLite 本地 / PostgreSQL（Neon）生产
- Prisma ORM
- NextAuth v5（Google + GitHub + Credential 演示登录）
- OpenAI 兼容 API（可切换 DeepSeek、商汤等）
- 部署：Vercel
